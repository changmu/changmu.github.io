javascript知识点总结

发表于 2022-06-10 更新于 2022-06-15 分类于 技术 ， javascript Waline： 本文字数： 1.1k 阅读时长 ≈ 1 分钟

‘use strict’;

  • 严格模式下，当一个函数声明在一个代码块内时，它在该代码块内的任何位置都是可见的。但在代码块外不可见。

比较

  • 在比较不同类型的值时，处于相等判断符号 == 两侧的值会先被转化为数字。空字符串和 false 也是如此，转化后它们都为数字 0。
  • 严格相等运算符 === 在进行比较时不会做任何的类型转换。

空值合并运算符 ??

空值合并运算符 ?? 是最近才被添加到 JavaScript 中的，它的出现是因为人们对 || 不太满意。
它们之间重要的区别是：

  • || 返回第一个 真 值。
  • ?? 返回第一个 已定义的 值。

数字类型

  • NaN === NaN // false
  • 判断数字是否有效isFinite
  • 生成指定范围内概率均匀的随机整数

数组

  • 取最后一个元素: fruits.at(-1)
  • 遍历数组使用for (let fruit of fruits)
  • 清空数组最简单的方法就是: arr.length = 0
  • 数组删除元素用arr.splice(start[, deleteCount, elem1, ..., elemN])
  • 拷贝数组arr.slice()
  • 拼接多个数组arr.concat()
  • 判断对象是否为数组Array.isArray()

对象

  • 对象解构赋值，函数参数默认值

函数

  • 箭头函数 this 的查找与常规变量的搜索方式完全相同：在外部词法环境中查找。
  • 普通函数this在调用时决定，即调用对象
  • 在一个对象还是在原型中。在一个方法调用中，this 始终是点符号 . 前面的对象。this 根本不受原型的影响。

ERD

ERD定义

ERD全称Entity Relationship Diagrams, 提供了表示存储结构的实体类型、属性和联系的方法，用来描述存储设计。

Markdown的mermaid插件提供了代码绘制ERD的功能，功能简洁，便于维护。

ERD概念

ERD的概念很简单，只需要将实体和实体间的关系描述清楚即可

实体Entity

实体(entity) 表示一个存储的对象，使用长方形表现

erDiagram
    Entity

实体可以使用表格形式将其属性包含进来

erDiagram
CAR {
    string allowedDriver FK "The license of the allowed driver"
    string registrationNumber
    string make
    string model
}

关联关系Relationship

实体间的基本关联关系有以下3种

1.一对一：对于实体集A与实体集B，A中的每一个实体至多与B中一个实体有关系；反之，在实体集B中的每个实体至多与实体集A中一个实体有关系。

erDiagram
    Citizen ||--|| IDnum: has

2.一对多：实体集A与实体集B中至少有N(N>0)个实体有关系；并且实体集B中每一个实体至多与实体集A中一个实体有关系。

erDiagram
    Class ||--|{ Student: has

3.多对多：实体集A中的每一个实体与实体集B中至少有M(M>0)个实体有关系，并且实体集B中的每一个实体与实体集A中的至少N（N>0）个实体有关系。

erDiagram
    Student }o--|{ Course: elective

mermaid对ERD的支持

ValueLeft ValueRight 含义
|o o| 0或1个
|| || 1个
}o o{ 0或多个
}| |{ 1或多个

demo app内测阶段激活码的ERD

erDiagram
Channel {
    int channel_id PK "渠道id"
    string name "渠道名"
}
InvitationCode {
    string code PK "邀请码"
    int channel_id FK "渠道id"
    int total "配额总个数"
    int used "已使用个数"
    int start_time "开始时间"
    int end_time "结束时间"
}
InvitationUser {
    string uid PK "被激活uid"
    string code "激活码"
    int ctime "激活时间"
}
Channel ||--o{ InvitationCode: has
InvitationCode ||--o{ InvitationUser: has
